# 新手版说明：这个自动审核系统现在是什么

## 1. 这套系统现在是什么

这是一套给 GitHub PR 用的“自动审代码”系统。

它现在已经能做到两件事：

1. 你开 PR 以后，系统会自动帮你审核 RTL / Verilog 代码
2. 如果你希望机器人帮你改，就在 PR 里写一句 `/codex-fix`，然后它会在这台电脑上自动改代码，再让 GPT 重新审核一次

你不用去手动运行脚本，也不用每次都重新配置。

一句话理解：

- **你开 PR**
- **GitHub 自动叫 GPT 来审**
- **你如果下令 `/codex-fix`，桌面版 Codex 会在本机改代码**
- **改完后 GitHub 再自动叫 GPT 复审**

---

## 2. 谁负责什么

### 2.1 GitHub 负责什么

GitHub 负责“自动触发流程”。

它会负责：

- 发现 PR 被创建或更新了
- 自动运行审核流程
- 把审核结果显示在 PR 页面上
- 当你写 `/codex-fix` 时，把这个请求挂到 PR 上

你可以把 GitHub 理解成“总开关”和“公告板”。

### 2.2 GPT-5.4 负责什么

GPT-5.4 负责“审代码”。

它会负责：

- 读取这次 PR 改了什么
- 检查 RTL / Verilog 是否有明显风险
- 把审核意见写回 PR 评论

它主要是审，不是本地直接改文件。

### 2.3 桌面版 Codex 负责什么

桌面版 Codex 负责“在这台电脑上改代码”。

它会负责：

- 读取 PR 上最新的审核意见
- 在本机打开对应分支
- 按规则修改代码
- 做最小验证
- commit 和 push

你可以把桌面版 Codex 理解成“本地修代码的工人”。

### 2.4 你负责什么

你平时只需要负责两件事：

1. 开 PR
2. 如果想让机器人修，就在 PR 里写 `/codex-fix`

其他事情都已经自动化了。

---

## 3. 你平时每次只需要做哪两步

以后你每次正常使用时，只做这两步：

### 第一步

正常提交代码并创建一个 PR。

### 第二步

看 PR 页面上的自动审核结果。

如果你希望机器人继续帮你修，就在 PR 评论里写：

```text
/codex-fix
```

你不需要再去碰脚本、工作流文件、模型配置。

---

## 4. 出问题先看哪里

如果你觉得“这次怎么没自动工作”，先按这个顺序看：

### 第一看：PR 页面里的评论区

先看有没有这条评论：

- `RTL Auto Review`

如果有，说明 GPT 审核已经跑了。

### 第二看：PR 页面里的标签

如果你写了 `/codex-fix`，就看 PR 有没有这些标签：

- `codex-fix-pending`
- `codex-fix-applied`
- `codex-fix-failed`

含义很简单：

- `codex-fix-pending`：已经排队，等桌面版 Codex 处理
- `codex-fix-applied`：已经修完并推送
- `codex-fix-failed`：修复失败，需要人工再看

### 第三看：GitHub Actions

如果 PR 没有出现自动审核评论，就去看仓库的 Actions 页面，重点看：

- `RTL PR Auto Review`
- `RTL PR Codex Fix Request`

通常问题都会在这里暴露出来。

## 5. 哪些配置已经固定好，你不要再碰

下面这些配置，这次已经验证通过了，**现在不要再随便改**：

- `OPENAI_API_BASE = https://hub.tokenpanda.top/v1`
- `OPENAI_REVIEW_MODEL = gpt-5.4`
- `OPENAI_ENDPOINT_STYLE = chat_completions`

还有这个 Secret 也已经在用：

- `OPENAI_API_KEY`

你现在不用再去折腾：

- 模型名
- base URL
- endpoint 类型
- workflow 触发条件
- GitHub 权限项

这些都已经调通了。

---

## 6. 当前系统已经验证到什么程度

这套闭环已经真实跑通过了，验收记录保留在测试 PR `#9`。

已经验证成功的内容是：

1. PR 创建后，GPT-5.4 会自动审核
2. PR 里写 `/codex-fix` 后，请求会自动进入队列
3. 桌面版 Codex 会在本机修代码并 push
4. push 之后，GPT-5.4 会自动再次审核

所以你现在可以把它当作“已经能用”的版本来使用。

剩下的流程，系统会自己跑。
